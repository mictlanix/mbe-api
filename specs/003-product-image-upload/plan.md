# Implementation Plan: Product Image Upload & Serving

**Branch**: `003-product-image-upload` | **Date**: 2026-06-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-product-image-upload/spec.md`

## Summary

Extends the already-implemented image upload feature (pillow, image_service, upload endpoint) with:
1. A public static-file serving endpoint at `/images/{filename}` (no auth required).
2. `images_base_url` config field so the API can construct absolute URLs.
3. `ProductResponse.photo` now returns a full URL (e.g. `http://host/images/{hash}.png`) instead of the bare filename, constructed at serialization time.

No new packages required ‚Äî FastAPI's bundled `StaticFiles` handles serving.

## Technical Context

**Language/Version**: Python 3.12+

**Primary Dependencies**:
- FastAPI `StaticFiles` (stdlib via starlette ‚Äî already installed) ‚Äî mounts `/images` as a static directory
- Pillow, image_service (existing from prior implementation)

**Storage**: Local filesystem, `settings.images_dir` (existing)

**Testing**: pytest + httpx ASGITransport (existing)

**Target Platform**: Linux server (FastAPI/ASGI)

**Project Type**: Web service ‚Äî endpoint addition + response shape change

**Constraints**:
- `ProductResponse.photo` must change from bare filename to full URL ‚Äî affects all endpoints that return `ProductResponse`
- `photo` column in DB continues to store bare filename; URL construction happens at read time
- `StaticFiles` with `check_dir=False` to allow startup without a pre-existing images dir
- Constitution VII: public serving endpoint MUST be explicitly marked public in spec ‚úì (FR-014, Assumptions)

**Scale/Scope**: One config field, one app-level mount, one helper function, one schema change, tests
